# 这是一个本地的 mcp server 的项目

- 代码中使用 McpServer 和 StdioServerTransport
- mcp 的配置是

```json
{
  "mcpServers": {
    "happy-arena-mcp": {
      "command": "npx",
      "args": ["-y", "happy-arena-mcp"]
    }
  }
}
```

## 如何和 builder 交互

- 比如用户在 builder 中提问，请根据这个链接地址帮我生成前端请求的代码，我的 mcp 服务中获取到这个地址来完成接下来第一步逻辑：根据用户提供的 url 比如http://127.0.0.1:8080/happy-arena-mcp/start/#/detail?id=121242，解析url中的信息，获取到url中的id，并调用getVersionDetial接口获取api的详细数据，返回data，当然这个地址是我摸你的地址，你不需要发送请求，只需要写出对应逻辑的代码

- 比如，用户说应用的 94345，名称是 aaa，根据用户提供的 appId 和接口名称查询 getApi 接口，获取 api 的详细数据返回 data

- 返回的 data 中包含了 operation 数据，这是标准规范的 openapi 格式的数据、

根据以上信息先完成第一步 mcp 的生成
